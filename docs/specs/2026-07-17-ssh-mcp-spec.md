# Spec：SSH MCP

> 状态：已确认（2026-07-17）。可作为后续技术设计与验收依据。

## 背景 / 范围

开发者越来越多地让 AI 完成开发与排障，但远程环境仍要求开发者手动打开 SSH、执行命令、传输文件并复制结果。本产品通过 MCP，让兼容客户端在用户控制下操作预先登记的开发或测试服务器，完成单机与少量多机任务闭环。

本规格覆盖 1–10 台已登记的 Linux 或 Windows 主机，包括主机发现与首次信任、低风险操作自动执行、风险审批、单次命令、持久交互终端、文件与目录传输、多主机协作、超时取消、部分失败及敏感信息保护。

本规格不覆盖生产环境、临时任意主机、凭据生命周期、多人权限、审计合规、大规模编排或 MCP 之外的产品入口。

### 功能空间调研说明

本规格在起草前完成了一轮 Web 功能空间调研，调研结果仅用于发现能力维度，没有自动转化为需求。主要一手来源：

- [MCP 规范](https://modelcontextprotocol.io/specification/2025-06-18/index)：工具调用、用户控制、进度、取消、错误与能力协商。
- [MCP Tasks](https://modelcontextprotocol.io/extensions/tasks/overview)：长任务、进度、中途输入、取消及客户端能力差异。
- [OpenSSH ssh(1)](https://man.openbsd.org/ssh.1) 与 [ssh_config(5)](https://man.openbsd.org/ssh_config.5)：命令会话、PTY、主机身份、超时、连接复用及转发边界。
- [OpenSSH sftp(1)](https://man.openbsd.org/sftp.1)：目录递归、进度、续传、权限与符号链接边界。
- [同类 SSH MCP 项目文档](https://github.com/classfang/ssh-mcp-server)：命名连接、文件传输、路径限制、命令限制、超时与结构化失败。

## 术语

- **登记主机**：由用户预先配置、具有唯一别名、环境类型、远程平台与既有 SSH 登录条件的主机。
- **低风险规则集**：用户在 MCP 会话外明确配置、允许无需逐次确认即可执行的只读操作集合；默认集合为空，AI 客户端不能修改。
- **审批**：客户端向用户展示完整操作信息后获得的一次性明确允许。
- **交互会话**：在一台主机上保持 Shell 状态、可持续收发终端输入输出的会话。
- **多主机操作**：一次操作显式指定 2–10 个唯一登记主机，并分别产生结果。

## 行为需求

### Requirement：MCP 行为与输入校验

系统 MUST 通过 MCP 暴露本规格纳入的能力，并为每项操作提供明确的输入约束与可观察结果；无效输入 MUST 在发起任何 SSH 或文件系统操作前被拒绝。

#### Scenario：发现可用能力

- WHEN 兼容 MCP 的客户端请求能力列表
- THEN 系统 MUST 返回本版支持的主机发现、命令执行、交互会话、文件传输、多主机操作及状态控制能力
- AND 每项能力 MUST 描述必填参数、允许值和结果结构

#### Scenario：拒绝额外或缺失参数

- WHEN 客户端提交缺少必填字段、字段类型错误或未声明字段的请求
- THEN 系统 MUST 返回参数错误
- AND MUST NOT 尝试连接主机、执行命令或读写文件

#### Scenario：客户端不能呈现所需审批

- WHEN 某项操作需要用户审批，但客户端不能完成审批交互
- THEN 系统 MUST 明确返回“客户端不支持所需审批”
- AND MUST NOT 启动该操作

### Requirement：登记主机边界

系统 MUST 只允许操作 1–10 台预先登记、具有唯一别名的开发或测试主机。每台主机 MUST 声明环境类型、远程平台和 Shell 类型。

#### Scenario：列出登记主机

- WHEN 客户端请求可用主机
- THEN 系统 MUST 按别名字典序稳定返回全部登记主机
- AND 每项只显示别名、环境类型、远程平台、Shell 类型及当前可观察连接状态
- AND MUST NOT 显示密码、私钥、口令、认证令牌或其派生值

#### Scenario：主机数量边界

- WHEN 配置包含 1 台或 10 台唯一登记主机
- THEN 系统 MUST 接受该配置
- WHEN 配置为空、包含重复别名或超过 10 台主机
- THEN 系统 MUST 拒绝进入可操作状态并明确指出配置问题

#### Scenario：拒绝未登记主机

- WHEN 操作请求包含未登记别名、IP 地址或临时连接信息
- THEN 系统 MUST 拒绝请求
- AND MUST NOT 对该地址发起网络连接

#### Scenario：拒绝生产环境登记

- WHEN 某主机的环境类型不是“开发”或“测试”
- THEN 系统 MUST 拒绝将其作为可操作主机

### Requirement：主机身份信任

系统 MUST 在认证前校验远程主机身份。首次遇到未知指纹时 MUST 请求用户确认；已信任主机的指纹发生变化时 MUST 严格拒绝连接。

#### Scenario：首次确认并建立信任

- WHEN 系统首次连接某登记主机且没有已信任指纹
- THEN 系统 MUST 在认证和执行前展示主机别名、地址及完整指纹并请求确认
- WHEN 用户明确接受该指纹
- THEN 系统 MUST 记住该信任关系并继续连接

#### Scenario：首次拒绝或无响应

- WHEN 用户拒绝首次指纹、审批超时或客户端无法呈现审批
- THEN 系统 MUST 终止连接
- AND MUST NOT 尝试认证或执行任何远程操作

#### Scenario：后续指纹一致

- WHEN 已登记主机返回与已信任值一致的指纹
- THEN 系统 MUST 继续认证且无需重复询问主机身份

#### Scenario：指纹发生变化

- WHEN 已登记主机返回的指纹与已信任值不同
- THEN 系统 MUST 拒绝连接并同时报告旧指纹与新指纹
- AND MUST NOT 提供在当前任务中临时绕过或覆盖该拒绝的入口

### Requirement：认证与敏感信息隔离

系统 MUST 仅使用用户事先准备的 SSH 登录条件，并 MUST 防止登录凭据通过 MCP 结果、审批内容或诊断信息暴露给 AI 客户端。

#### Scenario：使用既有登录条件

- WHEN 主机身份验证通过且登记主机的既有登录条件有效
- THEN 系统 MUST 完成认证并报告连接成功
- AND MUST NOT 要求 AI 提供密码、私钥内容或密钥口令

#### Scenario：认证失败

- WHEN 既有登录条件缺失、失效或需要未准备的交互式认证
- THEN 系统 MUST 返回明确的认证失败类别
- AND MUST NOT 在 MCP 会话中收集、保存或回显新的登录秘密

#### Scenario：错误和诊断脱敏

- WHEN 系统产生连接、认证、命令或传输错误
- THEN 对客户端可见的内容 MUST NOT 包含密码、私钥内容、密钥口令、认证令牌或完整凭据配置

### Requirement：操作授权

系统 MUST 仅自动执行完整匹配低风险规则集的只读操作。默认低风险规则集 MUST 为空；任何未匹配操作、交互输入、上传、下载或其他写入行为 MUST 在执行前取得一次性审批。

#### Scenario：自动执行明确允许的只读操作

- WHEN 操作的目标主机、远程平台、Shell 和完整操作内容均匹配一条低风险规则
- THEN 系统 MUST 将其标记为低风险并允许无需逐次审批执行
- AND 该规则 MUST 来自用户在 MCP 会话外作出的预先授权

#### Scenario：客户端尝试修改规则集

- WHEN AI 客户端请求新增、放宽、删除或临时覆盖低风险规则
- THEN 系统 MUST 拒绝请求
- AND 现有低风险规则集 MUST 保持不变

#### Scenario：未知或部分匹配的操作

- WHEN 操作未匹配规则、只匹配命令的一部分、通过组合命令追加了额外行为或无法确定风险
- THEN 系统 MUST 将整个操作视为需要审批
- AND MUST NOT 只自动执行其中看似低风险的部分

#### Scenario：审批信息完整

- WHEN 操作需要审批
- THEN 系统 MUST 展示全部目标主机、远程平台、精确命令或输入、文件源与目标、递归范围、覆盖行为及可识别的影响
- AND 在用户明确允许前 MUST NOT 执行任何远程或本地副作用

#### Scenario：审批绑定精确操作

- WHEN 用户批准一个操作
- THEN 该批准 MUST 只对所展示的主机集合和完整操作内容生效一次
- WHEN 主机集合、命令、交互输入、路径、递归范围或覆盖方式发生变化
- THEN 系统 MUST 请求新的审批

#### Scenario：拒绝或审批超时

- WHEN 用户拒绝审批、审批超时或审批通道中断
- THEN 系统 MUST 将操作标记为未执行
- AND MUST NOT 自动重试或降级执行

### Requirement：单次命令执行

系统 MUST 能在指定登记主机上执行单次远程命令，并返回足以区分远程程序结果与连接故障的结构化结果。

#### Scenario：命令成功

- WHEN 一个已授权命令在远程主机正常退出
- THEN 结果 MUST 包含主机别名、远程平台、标准输出、标准错误、退出状态和最终状态“完成”

#### Scenario：远程程序非零退出

- WHEN SSH 连接正常但远程程序以非零状态退出
- THEN 系统 MUST 返回该退出状态及已有输出
- AND MUST 将其标记为“远程程序失败”，而不是“SSH 连接失败”

#### Scenario：空命令

- WHEN 命令为空或只包含空白
- THEN 系统 MUST 在连接主机前拒绝请求

#### Scenario：输出超过可返回范围

- WHEN 命令输出超过系统声明的单次返回范围
- THEN 系统 MUST 明确标记输出被截断并报告已返回与未返回的数量
- AND MUST NOT 静默截断或错误宣称输出完整

#### Scenario：不同平台不做命令翻译

- WHEN 客户端向 Linux 或 Windows 主机提交命令
- THEN 系统 MUST 将命令交给该登记主机声明的 Shell
- AND MUST NOT 自动把 POSIX Shell 命令改写为 PowerShell 命令，或反向改写

### Requirement：交互会话

系统 MUST 能为单台登记主机建立带终端语义的持久交互会话，并在会话内保持远程 Shell 上下文。

#### Scenario：创建交互会话

- WHEN 客户端为已通过身份与认证校验的主机创建交互会话
- THEN 系统 MUST 返回唯一会话标识、主机别名、远程平台、Shell 类型和状态“活动”
- AND 会话 MUST 支持持续读取输出

#### Scenario：保持 Shell 上下文

- WHEN 客户端在同一会话中依次改变工作目录、设置会话环境并执行后续命令
- THEN 后续命令 MUST 观察到该会话此前建立的工作目录与 Shell 状态

#### Scenario：完整终端交互

- WHEN 已授权客户端向活动会话发送文本、按键、控制字符或终端尺寸变化
- THEN 远程终端 MUST 按顺序接收这些输入
- AND 客户端 MUST 能读取对应的后续终端输出

#### Scenario：输出游标与顺序

- WHEN 客户端多次读取同一会话的新增输出
- THEN 系统 MUST 保持远程接收顺序并提供不会静默重复或跳过输出的读取位置

#### Scenario：会话隔离

- WHEN 同一主机或不同主机存在多个活动会话
- THEN 每个会话的输入、输出、工作目录、环境和生命周期 MUST 相互隔离

#### Scenario：关闭会话的幂等性

- WHEN 客户端关闭一个活动会话
- THEN 系统 MUST 关闭对应终端通道并将其标记为“已关闭”
- WHEN 客户端再次关闭同一会话
- THEN 系统 MUST 返回“已关闭”且 MUST NOT 影响其他会话

#### Scenario：对失效会话操作

- WHEN 客户端向不存在、已关闭或已断开的会话发送输入
- THEN 系统 MUST 拒绝输入并返回当前会话状态
- AND MUST NOT 静默创建新会话或重放输入

### Requirement：长任务、超时与取消

系统 MUST 为长时间运行的命令、会话和文件传输提供可查询状态、增量进度、明确超时及取消能力，并 MUST NOT 无限等待或静默重试。操作状态 MUST 可区分“等待审批”“运行中”“完成”“失败”“已取消”“部分失败”和“状态未知”。

#### Scenario：查询长任务状态

- WHEN 操作尚未结束
- THEN 客户端 MUST 能查询其主机、当前状态、已产生的进度或输出以及最近一次状态变化

#### Scenario：操作超时

- WHEN 连接、命令、会话空闲或传输超过对应的已声明超时
- THEN 系统 MUST 尝试停止对应操作并返回“超时”
- AND MUST 保留超时前已经产生的可用输出或逐项结果

#### Scenario：取消并确认停止

- WHEN 客户端取消一个运行中的操作且系统确认远程操作已经停止
- THEN 系统 MUST 将其最终状态标记为“已取消”并停止产生新输出

#### Scenario：无法确认取消结果

- WHEN 系统无法确认远程进程或传输是否已停止
- THEN 系统 MUST 将状态标记为“状态未知”并给出诊断信息
- AND MUST NOT 错误报告为“已取消”或“已完成”

#### Scenario：不自动重试

- WHEN 连接中断、超时或产生可重试错误
- THEN 系统 MUST 标明错误是否可能重试
- AND MUST NOT 自动重新执行命令、交互输入或文件传输

### Requirement：文件与目录传输

系统 MUST 支持在允许的本地路径与登记主机允许的远程路径之间上传和下载单个文件或目录，并保持文件内容与目录相对结构。

#### Scenario：上传或下载单个文件

- WHEN 用户批准在允许路径之间传输一个文件且目标不存在
- THEN 系统 MUST 传输完整文件内容并报告源、目标、字节数和“完成”状态
- AND 二进制文件传输前后内容 MUST 完全一致

#### Scenario：递归传输目录

- WHEN 用户批准递归传输一个目录
- THEN 系统 MUST 在目标中保持源目录的相对层级和普通文件内容
- AND MUST 按文件报告进度或结果

#### Scenario：拒绝越界路径

- WHEN 本地或远程路径位于允许根目录之外，或通过 `..`、大小写差异、驱动器切换、链接或重解析点逃逸允许范围
- THEN 系统 MUST 拒绝传输
- AND MUST NOT 读取源内容或创建、修改目标内容

#### Scenario：目标已存在

- WHEN 传输目标已经存在且审批未明确包含覆盖
- THEN 系统 MUST 停止该目标的传输并返回“目标已存在”
- AND MUST NOT 静默覆盖、合并或重命名

#### Scenario：明确批准覆盖

- WHEN 审批明确展示并允许覆盖指定目标
- THEN 系统 MAY 替换该目标
- AND 传输失败时 MUST NOT 将不完整内容报告为成功

#### Scenario：遇到符号链接或重解析点

- WHEN 源目录或路径链包含符号链接、目录联接或其他重解析点
- THEN 系统 MUST 拒绝跟随该对象并返回明确错误
- AND MUST NOT 复制链接本身或访问链接指向范围

#### Scenario：目录传输部分失败

- WHEN 目录中的部分文件传输失败、超时或被取消
- THEN 系统 MUST 分别列出成功、失败和未执行项
- AND 整体状态 MUST 为“部分失败”或“已取消”，不得报告为完全成功

#### Scenario：传输进度

- WHEN 文件或目录传输正在进行
- THEN 客户端 MUST 能观察当前项、已传输字节数及已完成项数量

### Requirement：多主机协作

系统 MUST 支持一次操作显式指定 2–10 台唯一登记主机，并在各主机之间隔离执行状态和结果。

#### Scenario：并行执行多主机命令

- WHEN 客户端向 2–10 台唯一登记主机提交同一已授权命令并选择并行执行
- THEN 系统 MUST 对每台主机独立执行并持续返回带主机别名的状态与结果

#### Scenario：多主机顺序执行

- WHEN 客户端选择按显式顺序操作多台主机
- THEN 系统 MUST 按请求顺序启动各主机操作
- AND 每台主机的结果 MUST 保留其主机归属

#### Scenario：目标集合无效

- WHEN 多主机请求为空、只有 1 台、包含重复主机、包含未登记主机或超过 10 台
- THEN 系统 MUST 在任何主机执行前拒绝整个请求

#### Scenario：部分主机失败

- WHEN 某些主机连接、命令或传输失败
- THEN 系统 MUST 继续收集其他已启动主机的结果
- AND MUST 返回每台主机的独立最终状态以及整体“部分失败”状态
- AND MUST NOT 自动回滚已成功主机上的操作

#### Scenario：取消多主机操作

- WHEN 客户端取消一个多主机操作
- THEN 系统 MUST 停止启动尚未开始的主机操作，并分别尝试取消正在运行的主机操作
- AND 最终结果 MUST 逐台区分“已取消”“已完成”“失败”或“状态未知”

### Requirement：跨平台行为

系统 MUST 支持远程 Linux 的 POSIX Shell 与远程 Windows 的 PowerShell，并按登记平台处理终端、路径、换行与低风险规则。

#### Scenario：Linux 主机

- WHEN 登记平台为 Linux
- THEN 命令、允许路径和低风险规则 MUST 使用 Linux/POSIX 语义
- AND 交互会话 MUST 以登记的 POSIX Shell 行为为准

#### Scenario：Windows 主机

- WHEN 登记平台为 Windows
- THEN 命令和低风险规则 MUST 使用 PowerShell 语义
- AND 允许路径检查 MUST 正确处理盘符、分隔符与不区分大小写的路径比较

#### Scenario：平台声明与实际环境不兼容

- WHEN 主机实际 Shell 无法满足登记的平台与 Shell 声明
- THEN 系统 MUST 拒绝执行并返回“平台或 Shell 不兼容”
- AND MUST NOT 猜测或自动切换到另一种命令语言

#### Scenario：文本与二进制编码

- WHEN 终端输出包含中文、Unicode、不同换行风格或无效文本字节
- THEN 系统 MUST 保持内容与顺序；无法按文本解码的字节 MUST 以无损字节转义形式返回并明确标记
- WHEN 传输二进制文件
- THEN 系统 MUST 不进行字符编码或换行转换

### Requirement：结果与错误契约

系统 MUST 为所有操作返回稳定、可区分且不泄露秘密的结果；错误 MUST 包含类别、消息、关联主机或会话、最终状态及是否可能重试。

#### Scenario：连接类错误

- WHEN 发生主机未知、指纹拒绝、指纹变化、认证失败、连接拒绝或连接超时
- THEN 系统 MUST 返回彼此可区分的错误类别
- AND MUST 指明是否已对远程主机产生任何操作

#### Scenario：操作类错误

- WHEN 发生审批拒绝、路径越界、目标已存在、命令超时、会话失效、传输失败或部分失败
- THEN 系统 MUST 返回对应错误类别及已有的安全结果

#### Scenario：状态不可确定

- WHEN 网络中断导致系统无法判断远程操作是否仍在运行或是否已经产生副作用
- THEN 系统 MUST 返回“状态未知”
- AND MUST NOT 将其归类为可安全自动重试

## 不做什么（MUST NOT）

- 系统 MUST NOT 接受请求中临时提供的主机地址、账号、密码、私钥或密钥口令。验证：相关能力的输入约束不存在这些动态连接字段；提交时在网络连接前被拒绝。
- 系统 MUST NOT 允许 AI 客户端创建、修改、放宽或临时覆盖低风险规则集。验证：能力列表无规则写入入口；相关输入在产生副作用前被拒绝。
- 系统 MUST NOT 操作登记为生产环境或未明确标记为开发/测试的主机。验证：此类配置不能进入可操作主机列表。
- 系统 MUST NOT 提供账号开通、凭据保存、密钥生成、密钥轮换或 MFA 信息采集能力。验证：能力列表与审批流程中不存在相关入口。
- 系统 MUST NOT 提供多人角色、共享权限、长期审计记录或合规报表。验证：不存在用户、角色、租户、审计查询或导出能力。
- 系统 MUST NOT 接受主机通配符、动态主机组或超过 10 台的目标集合。验证：此类请求在执行前失败。
- 系统 MUST NOT 提供独立 CLI、HTTP 业务 API、网页或桌面管理界面。验证：用户可操作入口仅为 MCP。
- 系统 MUST NOT 创建或管理端口转发、动态代理、X11 转发、认证代理转发或隧道。验证：能力列表和输入中不存在相关操作；即使既有 SSH 配置包含这些请求，也不得由本产品主动开启。
- 系统 MUST NOT 在本版提供文件续传、保留权限/时间/所有者、复制符号链接或跟随重解析点。验证：中断后重新发起从头传输；结果不宣称元数据已保留；链接场景明确失败。
- 系统 MUST NOT 自动重试可能产生副作用的命令、输入或传输，也 MUST NOT 提供跨主机事务或自动回滚。验证：故障后调用次数不增加；成功主机结果保持原样并标记部分失败。
- 系统 MUST NOT 在主机指纹变化后提供任务内绕过方式。验证：即使用户尝试批准当前操作，连接仍被拒绝。
- 系统 MUST NOT 在 MCP 服务重启后宣称恢复此前的活动终端或运行任务。验证：旧会话标识在新生命周期中明确返回失效，不重放输入或命令。
- 系统 MUST NOT 在 Linux 与 Windows 之间自动翻译命令、路径或低风险规则。验证：平台不匹配时明确拒绝，而不是改写后执行。

## 边界场景覆盖（索引）

- **边界值** → Requirement「登记主机边界」Scenario「主机数量边界」；Requirement「多主机协作」Scenario「目标集合无效」。
- **空值** → Requirement「MCP 行为与输入校验」Scenario「拒绝额外或缺失参数」；Requirement「单次命令执行」Scenario「空命令」。
- **排序** → Requirement「登记主机边界」Scenario「列出登记主机」。
- **并发** → Requirement「交互会话」Scenario「会话隔离」；Requirement「多主机协作」Scenario「并行执行多主机命令」。
- **幂等** → Requirement「交互会话」Scenario「关闭会话的幂等性」；Requirement「操作授权」Scenario「审批绑定精确操作」。
- **精度** → Requirement「文件与目录传输」Scenario「上传或下载单个文件」「传输进度」；字节数必须按实际字节报告。金额、浮点业务精度不适用。
- **编码** → Requirement「跨平台行为」Scenario「文本与二进制编码」；Requirement「文件与目录传输」Scenario「上传或下载单个文件」。
- **超时与失败** → Requirement「长任务、超时与取消」全部 Scenario；Requirement「多主机协作」Scenario「部分主机失败」；Requirement「结果与错误契约」全部 Scenario。
- **路径逃逸与链接** → Requirement「文件与目录传输」Scenario「拒绝越界路径」「遇到符号链接或重解析点」。
- **审批中断** → Requirement「操作授权」Scenario「拒绝或审批超时」；Requirement「MCP 行为与输入校验」Scenario「客户端不能呈现所需审批」。
- **主机身份变化** → Requirement「主机身份信任」Scenario「指纹发生变化」。
- **禁止项** → 见「不做什么（MUST NOT）」逐条验证。

## 成功标准

- 自动化验收套件 MUST 覆盖本规格每个 Scenario，并由仓库提供一个统一测试命令；该命令在干净环境中退出状态为 0。
- MCP 协议验收 MUST 能发现全部纳入能力，拒绝非法参数，并验证需要审批的操作在批准前没有产生网络、远程或本地副作用。
- Linux 验收环境 MUST 完成：首次指纹确认、低风险只读命令、需审批命令、持久 POSIX Shell、中文输出、二进制文件、递归目录与超时取消。
- Windows 验收环境 MUST 完成：首次指纹确认、低风险 PowerShell 命令、需审批命令、持久 PowerShell、盘符路径、大小写路径边界、二进制文件、递归目录与超时取消。
- 多主机验收 MUST 同时覆盖 2 台成功、10 台边界、11 台拒绝，以及至少 1 台失败时的逐主机结果与整体部分失败。
- 安全验收 MUST 证明：未登记主机不会产生连接；指纹变化不能被任务内绕过；越界路径不会被读取或写入；拒绝审批不会执行；客户端可见结果不包含预置凭据。
- 文件验收 MUST 证明：单文件与目录传输内容一致、目标存在时不静默覆盖、链接不被跟随、部分失败不被报告为成功。
- 生命周期验收 MUST 证明：输出增量读取不静默重复或跳过；关闭会话可重复；取消结果不会在未确认停止时被误报为已取消；服务重启后旧会话不会被恢复或重放。

## 采用的保守默认

- 低风险规则集默认为空；未经用户明确配置时，所有操作均需要审批。
- 多主机部分失败时继续收集已启动主机结果，不自动回滚成功主机。
- 任何无法判断风险、主机状态、远程副作用或取消结果的情况都按更严格状态处理。
- 客户端不支持审批或必要交互时，操作失败关闭，不静默降级。
- 文件与目录传输不跟随链接、不保留元数据、不续传、不静默覆盖。
