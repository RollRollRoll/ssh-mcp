# Spec：SSH MCP 本机实例控制台

> 状态：已确认（2026-07-22）。可作为后续技术设计与验收依据。

## 背景 / 范围

当前 SSH MCP 仅通过 `stdio` 与 MCP 客户端通信。仓库中的前端是使用演示数据的独立原型，无法反映某个真实 MCP 进程的状态，也不能安全地控制该进程。

本规格新增一个与单个 SSH MCP 进程生命周期绑定的本机控制台。控制台用于查看该实例的主机、连接、操作、会话摘要、传输进度和审批状态；允许在网页中向一台登记主机执行单次命令或预设 Profile、取消可取消操作，并与 MCP 客户端并行处理由 MCP 调用产生的审批。

控制台仅供持有该进程专属访问凭证的本机用户使用。它复用 SSH MCP 已有的主机登记、主机身份信任、认证、策略、审批、超时、取消、输出限制和敏感信息保护边界，不形成第二套 SSH 执行规则。

本规格不增加浏览器内交互终端、文件上传下载、远程访问、多人权限、跨实例聚合或进程退出后的历史保留。

### 与既有规格的关系

本规格经确认后，修改 [SSH MCP 主规格](./2026-07-17-ssh-mcp-spec.md) 的以下既有约束：

- “系统 MUST NOT 提供独立 CLI、HTTP 业务 API、网页或桌面管理界面”改为：仅允许本规格定义的、与单个进程绑定的本机网页控制入口；独立业务 API、桌面界面和远程管理界面仍不在范围内。
- “客户端不能呈现所需审批时，操作失败关闭”改为：若该操作已进入本机控制台的待审批列表，则控制台可以在同一审批期限内处理；两个通道均不可用、拒绝、取消或超时时才失败关闭。

主规格的其余安全边界与行为要求保持不变。

### 功能空间调研说明

起草前使用公开的一手资料进行了功能空间调研。调研结果只用于发现能力维度，没有自动转化为需求：

- [MCP Elicitation 规范](https://modelcontextprotocol.io/specification/2025-11-25/client/elicitation)：审批的接受、拒绝、取消语义，审批来源标识、能力差异与失败处理。
- [MCP Cancellation 规范](https://modelcontextprotocol.io/specification/2025-11-25/basic/utilities/cancellation)：取消与完成竞态、取消请求的可观察状态及未知请求处理。
- [Cockpit 官方文档](https://cockpit-project.org/guide/latest/)：本机管理页面的会话隔离、访问保护、进程生命周期和本地无认证端口风险。
- [OWASP WebSocket Security Cheat Sheet](https://cheatsheetseries.owasp.org/cheatsheets/WebSocket_Security_Cheat_Sheet.html)：实时连接的来源校验、跨站劫持防护、会话失效和资源限制。
- [MDN Secure Contexts](https://developer.mozilla.org/en-US/docs/Web/Security/Defenses/Secure_Contexts)：浏览器对 `localhost` 与回环地址的安全上下文边界。

## 术语

- **实例**：一次运行中的 SSH MCP 进程及其内存状态；不同进程是不同实例。
- **本机控制台**：只允许从回环网络访问、只控制一个实例的网页入口。
- **实例访问凭证**：实例启动时产生、实例退出时失效的不可猜测凭证；持有者可查看并控制该实例。
- **MCP 来源操作**：由 MCP 客户端调用工具发起的操作。
- **网页来源操作**：由持有有效实例访问凭证的用户在本机控制台中发起的操作。
- **待审批项**：完整操作意图已冻结、尚未获得接受、拒绝、取消或超时结果的操作。
- **首个决定生效**：同一待审批项收到多个并发决定时，系统只接受最先完成状态转换的一个决定，其余决定不产生副作用。

## 行为需求

### Requirement：实例启动与关闭

系统 MUST 默认只启动 MCP `stdio`。客户端首次调用工具且支持 form elicitation 时，系统 MUST 只询问一次是否启用独立的本机控制台；只有用户接受后才尝试启动控制台。控制台一旦启用，MUST 与该实例共享生命周期。MCP `stdio` 标准输出 MUST 继续只承载 MCP JSON-RPC 帧。

#### Scenario：成功启动实例控制台

- WHEN SSH MCP 配置有效、实例完成 `stdio` 启动，且用户在首次工具调用中接受启用控制台
- THEN 系统 MUST 在不重启 MCP 的情况下尝试使该实例的本机控制台进入可用状态
- AND MUST 在诊断输出中给出该实例唯一的控制台访问地址
- AND 客户端声明支持 URL elicitation 时，系统 MUST 再请求客户端向用户呈现一次该地址的导航确认，并只在用户接受后由客户端打开
- AND 系统 MUST NOT 把包含实例访问凭证的地址附加到普通工具结果
- AND 访问地址 MUST 对应当前实例，不能展示或控制其他实例

#### Scenario：多个实例同时运行

- WHEN 同一台机器同时启动两个或更多 SSH MCP 实例
- THEN 每个实例 MUST 输出不同且可独立访问的控制台地址
- AND 任一控制台 MUST NOT 展示、审批、取消或执行其他实例的操作

#### Scenario：控制台启动失败

- WHEN 用户接受启用控制台，但系统无法建立满足本规格访问边界的控制台
- THEN MCP `stdio` 实例 MUST 继续可用，原工具调用 MUST 继续执行
- AND MUST 通过诊断输出及首次工具结果报告明确的控制台启动失败
- AND MUST 使用稳定的控制台启动错误码，并在安全白名单内保留可用的操作系统错误码
- AND MUST NOT 污染 MCP `stdio` 标准输出

#### Scenario：拒绝启用或客户端不支持询问

- WHEN 用户拒绝或取消启用控制台，或客户端不支持 form elicitation
- THEN 当前进程 MUST 保持纯 MCP `stdio`
- AND MUST NOT 加载控制台资产或建立 HTTP listener
- AND 后续工具调用 MUST NOT 重复询问

#### Scenario：实例正常退出

- WHEN SSH MCP 实例开始关闭或已经退出
- THEN 该实例的控制台 MUST 停止接受新的查看、审批、取消或执行请求
- AND 已打开页面 MUST 明确显示实例已离线或连接已结束，不能继续显示为在线

#### Scenario：实例异常终止

- WHEN SSH MCP 进程异常退出
- THEN 控制台连接 MUST 随进程失效
- AND 刷新旧地址 MUST NOT 恢复旧实例状态或将其连接到后来启动的新实例

### Requirement：本机访问保护

系统 MUST 只允许通过本机回环网络访问控制台，并要求每个受保护请求关联当前实例有效的访问凭证。访问凭证 MUST 只在当前实例生命周期内有效。

#### Scenario：持有有效凭证的本机访问

- WHEN 浏览器从本机回环地址访问当前实例，并提交该实例有效的访问凭证
- THEN 系统 MUST 允许其查看状态和使用本规格纳入的控制能力
- AND MUST NOT 再要求用户名或密码登录

#### Scenario：缺少、错误或属于其他实例的凭证

- WHEN 请求未携带访问凭证、凭证错误或凭证属于另一个实例
- THEN 系统 MUST 拒绝请求且不返回实例状态、主机信息、审批内容或命令输出
- AND MUST NOT 创建、审批、取消或修改任何操作

#### Scenario：非回环地址访问

- WHEN 请求通过局域网地址、外部网络接口或非本机来源尝试访问控制台
- THEN 系统 MUST 拒绝访问
- AND MUST NOT 因配置、请求头或名称解析结果而放宽为远程访问

#### Scenario：不可信网页尝试调用本机控制台

- WHEN 不属于当前控制台页面来源的网页尝试读取状态、建立实时连接或提交有副作用请求
- THEN 系统 MUST 拒绝该请求
- AND MUST NOT 仅凭请求来自浏览器或目标地址为 `localhost` 就信任它

#### Scenario：凭证随实例失效

- WHEN 实例退出后使用其旧访问凭证访问旧地址或后来启动的实例
- THEN 系统 MUST 拒绝该凭证
- AND MUST NOT 恢复、迁移或复用旧实例权限

### Requirement：实例状态总览

控制台 MUST 展示当前实例自身的服务状态、登记主机安全摘要、活动操作数量、待审批数量，以及 MCP 已创建的活动会话和传输摘要。状态展示 MUST 来源于实例的真实内存状态，不得使用演示数据推断在线状态。

#### Scenario：实例处于活动状态

- WHEN 用户使用有效凭证打开控制台且 MCP 实例正在运行
- THEN 页面 MUST 明确展示当前实例在线
- AND MUST 展示登记主机数量、活动操作数量和待审批数量

#### Scenario：展示登记主机

- WHEN 当前实例已加载一台或多台登记主机
- THEN 页面 MUST 按稳定顺序展示每台主机的别名、环境类型、平台、Shell 类型和已观察连接状态
- AND MUST NOT 为了刷新状态而主动建立新的 SSH 连接

#### Scenario：没有活动状态

- WHEN 当前没有运行中操作、待审批项、活动会话或传输
- THEN 页面 MUST 展示明确空态
- AND MUST NOT 伪造示例活动、成功记录或在线主机

#### Scenario：只读展示既有会话和传输

- WHEN MCP 客户端创建了交互会话或文件传输
- THEN 页面 MUST 能展示其安全状态摘要及关联主机
- AND 对文件传输 MUST 展示实例已有的进度与最终状态
- AND MUST NOT 提供会话输入、终端尺寸修改、上传或下载入口

### Requirement：操作状态与输出

控制台 MUST 展示当前实例中仍处于保留期内的操作，并准确反映等待审批、运行中、完成、失败、已取消、部分失败、超时和状态未知等状态。

#### Scenario：展示操作列表

- WHEN 实例中存在一个或多个仍处于保留期内的操作
- THEN 页面 MUST 展示操作标识、来源、目标主机、操作类型、当前状态和最近状态变化时间
- AND 相同时间条件下 MUST 使用稳定次序，刷新页面后不能随机重排

#### Scenario：查看运行中命令输出

- WHEN 命令正在产生标准输出或标准错误
- THEN 页面 MUST 保持实例接收顺序展示可用输出
- AND 新输出 MUST 在无需手动刷新页面的情况下出现
- AND 标准输出与标准错误 MUST 可区分

#### Scenario：输出被截断或已经过期

- WHEN 实例的输出缓冲区已经丢弃早期内容或操作记录超过保留期
- THEN 页面 MUST 明确标记输出被截断或记录已过期
- AND MUST NOT 将不完整内容宣称为完整输出
- AND MUST NOT 从磁盘或其他实例恢复该内容

#### Scenario：状态不可确定

- WHEN 实例无法确认远程副作用、取消结果或连接终止后的最终状态
- THEN 页面 MUST 展示“状态未知”及安全诊断摘要
- AND MUST NOT 改写为“成功”或“已取消”

### Requirement：网页发起操作

持有有效实例访问凭证的用户 MUST 能从控制台选择一台登记主机，提交单次命令或选择已配置的 Profile。网页来源操作 MUST 复用实例已有的主机信任、认证、策略、超时、输出和错误边界。

#### Scenario：预览并确认单次命令

- WHEN 用户选择一台登记主机并提交非空命令
- THEN 页面 MUST 在执行前展示目标主机、平台、完整命令、可识别影响和绑定该精确意图的摘要
- AND 用户明确确认前 MUST NOT 连接主机或执行命令
- WHEN 用户确认未发生变化的完整意图
- THEN 系统 MUST 只执行该意图一次

#### Scenario：预览并确认 Profile

- WHEN 用户选择一台登记主机和一个适用于该主机的已配置 Profile
- THEN 页面 MUST 在执行前展示目标主机、Profile 名称、实际完整操作内容和影响摘要
- AND 用户明确确认前 MUST NOT 执行 Profile

#### Scenario：预览后修改操作

- WHEN 命令、Profile、目标主机或任何影响执行结果的字段在预览后发生变化
- THEN 旧确认 MUST 失效
- AND 页面 MUST 基于新内容重新展示完整预览并取得新确认

#### Scenario：取消网页确认

- WHEN 用户关闭确认界面、选择取消或在确认期限内未作决定
- THEN 系统 MUST NOT 执行该网页来源操作
- AND MUST 清楚区分“用户取消”与远程执行失败

#### Scenario：拒绝无效网页输入

- WHEN 命令为空或只含空白、目标主机未登记、Profile 不存在或不适用于目标主机
- THEN 页面 MUST 在产生任何 SSH 副作用前拒绝提交并指出无效字段

#### Scenario：保持命令内容与平台语义

- WHEN 命令包含中文、Unicode、引号、换行或 Shell 元字符
- THEN 预览内容与实例最终交给登记 Shell 的内容 MUST 完全一致
- AND 系统 MUST NOT 在 Linux 与 Windows 之间翻译或改写命令

#### Scenario：网页来源操作不打扰 MCP 客户端

- WHEN 用户在网页中发起命令或 Profile
- THEN 审批 MUST 只在网页内通过完整预览和二次确认完成
- AND 系统 MUST NOT 为该网页来源操作向 MCP 客户端发起无对应工具调用上下文的审批请求

### Requirement：双通道审批

MCP 来源操作需要审批时，系统 MUST 将同一个冻结的待审批项同时提供给 MCP 客户端和本机控制台；任一通道的首个有效决定 MUST 成为唯一结果。

#### Scenario：两端展示同一审批

- WHEN MCP 来源操作进入等待审批状态
- THEN MCP 客户端与持有有效凭证的控制台 MUST 能看到相同的目标主机、平台、完整操作、影响摘要、审批期限和意图摘要
- AND 两端展示的内容 MUST 绑定同一个待审批项

#### Scenario：网页先批准

- WHEN 网页首先接受某个待审批项
- THEN 系统 MUST 只执行被冻结的精确操作一次
- AND MCP 客户端中的对应审批 MUST 结束或明确显示已由另一通道处理

#### Scenario：MCP 客户端先批准

- WHEN MCP 客户端首先接受某个待审批项
- THEN 系统 MUST 只执行被冻结的精确操作一次
- AND 网页中的对应审批 MUST 立即变为已处理且不能再次提交

#### Scenario：拒绝、取消与接受发生竞争

- WHEN 网页、其他网页标签页和 MCP 客户端并发提交接受、拒绝或取消决定
- THEN 系统 MUST 只接受首个完成状态转换的有效决定
- AND 后续决定 MUST 返回或显示“已处理”，不得改变最终决定或产生第二次副作用

#### Scenario：网页拒绝或取消 MCP 来源审批

- WHEN 网页首先拒绝待审批项
- THEN 系统 MUST 将其作为明确拒绝处理且 MUST NOT 执行操作
- WHEN 网页首先取消待审批项
- THEN 系统 MUST 将其作为取消处理且 MUST NOT 执行操作
- AND MCP 客户端 MUST 能区分拒绝、取消、超时和通道断开

#### Scenario：MCP 客户端不支持审批但网页可用

- WHEN MCP 客户端不能呈现所需审批，而待审批项可由本机控制台处理
- THEN 系统 MUST 在既有审批期限内保留该待审批项供网页决定
- AND MUST NOT 仅因 MCP 客户端缺少审批能力就提前执行或失败

#### Scenario：所有审批通道均无有效决定

- WHEN MCP 客户端不能处理审批，且网页未在审批期限内作出有效决定
- THEN 系统 MUST 将操作标记为审批超时或审批失败
- AND MUST NOT 执行操作、自动重试或降级绕过审批

#### Scenario：审批内容在等待期间发生变化

- WHEN 待审批项对应的主机、命令、Profile 内容或其他执行字段在等待期间无法保持原值
- THEN 系统 MUST 使原待审批项失效
- AND MUST NOT 使用旧批准执行变化后的操作

### Requirement：网页取消操作

控制台 MUST 允许用户对当前实例中支持取消且尚未进入终态的操作请求取消。取消是停止请求，不是完成保证。

#### Scenario：请求取消运行中操作

- WHEN 用户使用有效凭证取消一个运行中且可取消的操作
- THEN 页面 MUST 立即显示“已请求取消”
- AND 系统 MUST 尝试停止该操作并等待最终状态

#### Scenario：确认操作已经停止

- WHEN 系统确认远程操作因取消而停止
- THEN 页面 MUST 展示最终状态“已取消”
- AND MUST 停止将后续输出归入该操作

#### Scenario：取消与正常完成竞争

- WHEN 用户请求取消时操作已经完成或在取消生效前正常完成
- THEN 页面 MUST 展示真实最终状态“完成”或“失败”
- AND MUST NOT 因用户点击过取消而将其改写为“已取消”

#### Scenario：无法确认停止

- WHEN 取消确认期限结束后仍无法判断远程操作是否停止
- THEN 页面 MUST 展示“状态未知”及相应诊断摘要
- AND MUST NOT 宣称操作已安全停止

#### Scenario：重复取消

- WHEN 一个或多个网页标签页重复取消同一操作
- THEN 系统 MUST NOT 重复启动操作、影响其他操作或产生相互矛盾的最终状态
- AND 页面 MUST 返回当前取消或最终状态

#### Scenario：取消不允许取消的对象

- WHEN 用户尝试取消不存在、已经终结、属于其他实例或本身不可取消的操作
- THEN 系统 MUST 拒绝请求或返回其既有终态
- AND MUST NOT 影响任何其他操作

### Requirement：实时同步与恢复

控制台 MUST 在页面打开期间自动同步当前实例的状态变化，并在短暂连接中断后重新取得权威快照。页面不得在状态未知时允许基于陈旧信息产生副作用。

#### Scenario：状态自动更新

- WHEN 主机连接、操作、输出、进度或审批状态发生变化
- THEN 所有持有有效凭证并连接当前实例的页面 MUST 在无需手动刷新时观察到该变化
- AND 各页面最终 MUST 收敛到相同状态

#### Scenario：刷新或重新打开页面

- WHEN 用户使用同一有效访问地址刷新或重新打开控制台
- THEN 页面 MUST 从当前实例取得权威快照
- AND MUST 展示仍在保留期内的状态，不能恢复为演示数据或仅显示刷新后的新事件

#### Scenario：实时连接暂时中断

- WHEN 页面与实例的实时连接中断但页面仍打开
- THEN 页面 MUST 明确显示连接中断
- AND MUST 暂停审批、取消和执行入口，直到重新验证凭证并完成状态同步

#### Scenario：重连时遗漏了事件

- WHEN 页面断线期间操作或审批状态已经变化
- THEN 重连后的权威快照 MUST 覆盖页面旧状态
- AND 页面 MUST NOT 重放已经完成的审批、取消或执行请求

### Requirement：安全展示与输入处理

控制台 MUST 只展示完成其功能所需的数据，并对来自配置、远程输出、错误和用户输入的内容采用与 MCP 输出一致或更严格的敏感信息保护与安全渲染规则。

#### Scenario：主机与配置摘要脱敏

- WHEN 页面展示实例或主机信息
- THEN 页面 MUST NOT 展示密码、私钥、密钥口令、认证令牌、实例访问凭证或完整凭据配置
- AND 主机字段 MUST 不超过 MCP `hosts_list` 已允许的安全摘要，除非后续规格另行批准

#### Scenario：命令输出与错误脱敏

- WHEN 页面展示命令输出、连接错误、审批错误或诊断信息
- THEN 已知凭据和受保护配置值 MUST 按实例既有规则脱敏
- AND 页面 MUST NOT 将这些内容发送到外部服务

#### Scenario：不执行输出中的页面代码

- WHEN 命令、主机字段、远程输出或错误包含 HTML、脚本、事件属性或链接文本
- THEN 页面 MUST 将其作为数据展示
- AND MUST NOT 执行其中的代码、自动打开链接或改变控制台权限

#### Scenario：达到实例资源限制

- WHEN 网页请求会超过实例已有的活动操作数、记录数、输出缓冲或其他资源限制
- THEN 系统 MUST 使用与 MCP 调用一致的资源限制错误拒绝请求
- AND MUST NOT 为网页入口放宽限制或挤占后绕过限制

### Requirement：控制台可用性

控制台 MUST 提供中文界面、清晰的实例与连接状态，并使核心查看、确认、拒绝、取消和执行流程可通过键盘完成。

#### Scenario：审批对话框的键盘与焦点行为

- WHEN 用户通过键盘打开审批或网页操作确认界面
- THEN 焦点 MUST 进入对话框并停留在其交互范围内
- AND 用户 MUST 能通过键盘查看完整内容、接受、拒绝或取消
- AND 关闭后焦点 MUST 返回触发控件

#### Scenario：状态不只依赖颜色

- WHEN 页面展示在线、等待审批、运行、成功、失败、取消或断线状态
- THEN 每种状态 MUST 同时具有可读文字
- AND MUST NOT 仅以颜色或动画表达关键区别

#### Scenario：执行危险操作前可完整检查

- WHEN 命令或审批内容超过当前可视区域
- THEN 用户 MUST 能在作出决定前查看完整内容
- AND 主要决定控件 MUST 清楚区分接受、拒绝和取消

## 不做什么（MUST NOT）

- 系统 MUST NOT 在回环地址以外监听或接受控制台请求。验证：从局域网地址和非回环接口连接均失败，且不存在页面内开启远程访问的入口。
- 系统 MUST NOT 把“仅本机”当作访问授权，缺少有效实例访问凭证时不得返回状态或执行动作。验证：无凭证、错误凭证、跨实例凭证及不可信网页来源请求均无权访问。
- 系统 MUST NOT 提供用户名/密码登录、多人角色、共享权限或访问凭证管理页面。验证：页面不存在用户、角色、登录和凭证签发入口。
- 系统 MUST NOT 直接调用系统命令或浏览器 API 自动打开页面；只允许客户端在标准 URL elicitation 中取得用户同意后导航。验证：服务端无系统浏览器依赖或子进程调用，缺少 URL elicitation 能力时仅输出诊断地址。
- 系统 MUST NOT 提供浏览器内交互终端、会话输入、会话尺寸调整或从网页创建交互会话。验证：页面无终端和相应操作入口；既有会话只读展示摘要。
- 系统 MUST NOT 提供网页文件上传、下载、目录传输或覆盖操作。验证：页面无文件选择、拖放、上传或下载入口；既有传输只读展示状态。
- 系统 MUST NOT 在本版从网页一次向多台主机提交命令或 Profile。验证：每个网页来源操作只能选择一个登记主机。
- 系统 MUST NOT 聚合、发现或控制其他 SSH MCP 实例。验证：页面无实例扫描、切换或汇总入口，跨实例标识和凭证请求失败。
- 系统 MUST NOT 在实例退出后持久化或恢复操作、输出、审批、会话摘要或传输状态。验证：新实例无法查询旧实例状态，旧凭证失效。
- 系统 MUST NOT 提供长期审计记录、历史导出、外部通知、遥测或向第三方发送状态。验证：页面无相关入口，运行期间不产生为这些能力服务的外部请求。
- 系统 MUST NOT 让网页来源操作绕过登记主机、主机身份信任、认证、策略、审批、超时、取消、输出或敏感信息保护规则。验证：同一意图经 MCP 和网页进入执行核心时得到相同安全判定。
- 系统 MUST NOT 将控制台诊断、访问地址或其他文本作为非协议内容写入 MCP `stdio` 标准输出，也不得把访问地址放进普通工具结果；标准 URL elicitation 请求不受此限。验证：标准输出只包含合法 MCP JSON-RPC 帧，工具结果不含实例访问凭证。

## 边界场景覆盖（索引）

- **边界值** → Requirement「网页发起操作」Scenario「拒绝无效网页输入」；Requirement「安全展示与输入处理」Scenario「达到实例资源限制」。
- **空值** → Requirement「实例状态总览」Scenario「没有活动状态」；Requirement「网页发起操作」Scenario「拒绝无效网页输入」。
- **排序** → Requirement「实例状态总览」Scenario「展示登记主机」；Requirement「操作状态与输出」Scenario「展示操作列表」。
- **并发** → Requirement「双通道审批」Scenario「拒绝、取消与接受发生竞争」；Requirement「网页取消操作」Scenario「取消与正常完成竞争」；Requirement「实例启动与关闭」Scenario「多个实例同时运行」。
- **幂等** → Requirement「双通道审批」Scenario「拒绝、取消与接受发生竞争」；Requirement「网页取消操作」Scenario「重复取消」；Requirement「实时同步与恢复」Scenario「重连时遗漏了事件」。
- **精度** → Requirement「实例状态总览」Scenario「只读展示既有会话和传输」使用实例已有的字节与进度值；金额与浮点业务精度不适用。
- **编码** → Requirement「网页发起操作」Scenario「保持命令内容与平台语义」；Requirement「安全展示与输入处理」Scenario「不执行输出中的页面代码」。
- **超时与失败** → Requirement「实例启动与关闭」Scenario「控制台启动失败」「实例异常终止」；Requirement「双通道审批」Scenario「所有审批通道均无有效决定」；Requirement「网页取消操作」Scenario「无法确认停止」；Requirement「实时同步与恢复」全部 Scenario。
- **访问边界** → Requirement「本机访问保护」全部 Scenario。
- **禁止项** → 见「不做什么（MUST NOT）」逐条验证。

## 成功标准

- 自动化验收 MUST 启动一个真实测试实例，从诊断输出取得其控制台地址，验证页面展示的实例、主机和操作状态来自该实例而非演示数据；测试结束后旧地址与旧凭证均失效。
- 多实例验收 MUST 同时启动至少两个实例，证明两者地址、凭证、状态、审批和操作完全隔离。
- 访问安全验收 MUST 证明：非回环访问、无凭证、错误凭证、跨实例凭证和不可信网页来源请求均不能读取状态或产生副作用。
- 网页操作验收 MUST 覆盖：空命令拒绝、完整预览、预览后修改需重新确认、取消确认不执行、确认后精确执行一次、Profile 执行，以及 Unicode 和 Shell 元字符保持不变。
- 双通道审批验收 MUST 分别覆盖网页先决定、MCP 客户端先决定、客户端不支持审批时网页决定，以及接受/拒绝/取消并发竞争；每种竞争都只能得到一个最终决定和至多一次副作用。
- 取消验收 MUST 覆盖运行中取消成功、取消与正常完成竞争、重复取消和无法确认停止，且页面不能把“已请求取消”误报为最终“已取消”。
- 同步验收 MUST 覆盖多标签页、刷新、短暂断线及断线期间状态变化；恢复后所有页面收敛到实例权威状态，断线期间不能提交有副作用动作。
- 安全展示验收 MUST 证明：主机配置、错误、命令输出和审批内容不会泄露已知凭据；HTML 或脚本样式输出只作为文本显示；页面不会向外部服务发送运行状态。
- 协议验收 MUST 证明控制台启动与运行不会向 MCP `stdio` 标准输出混入任何非 JSON-RPC 内容。
- 无障碍验收 MUST 覆盖审批对话框焦点、全键盘决定流程、可读标签以及不只依赖颜色的状态表达。
- 仓库 MUST 提供统一的自动化验证入口，并使现有 MCP 单元、契约、验收测试与控制台构建及行为测试全部通过。

## 采用的保守默认

- 控制台是按需能力；无法安全启动控制台时，实例保持纯 `stdio` 可用，且不得自行提升或绕过宿主沙箱权限。
- 网页来源命令和 Profile 一次只面向一台登记主机，并且都需要网页内完整预览和二次确认。
- MCP 来源审批在网页和 MCP 客户端之间采用首个决定生效；后续决定只观察结果，不改变结果。
- 控制台只展示实例已经观察到的连接状态，不为刷新页面主动探测或连接主机。
- 完成操作只在实例现有内存保留期内可见，不增加磁盘持久化或浏览器持久化。
- 实时连接断开或状态未完成同步时，所有可能产生副作用的网页入口暂停使用。

## 待解问题

以下问题不阻塞行为规格成立，但必须在技术设计或实现前确定：

- 控制台访问地址、实例访问凭证和实时同步通道的具体表达方式。
- 多实例并存时访问地址的分配与冲突处理方式。
- 控制台静态资源如何随 SSH MCP 构建产物分发，以及现有 `web/` 原型中哪些内容保留。
- 实例状态快照与增量事件的内部契约，以及断线后的同步策略。
- 双通道审批如何对现有 MCP Elicitation 的待处理请求完成仲裁和通知。
- 浏览器支持范围，以及自动化测试采用的浏览器与无障碍检查工具。
